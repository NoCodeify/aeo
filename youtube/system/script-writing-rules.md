# Script Writing Rules

Operational reference for the script-writer agent. Read this file before writing any script.

---

## Three Non-Negotiable Rules

1. **Anticipation mechanics are sacred** - cannot be "improved away" in the name of voice
2. **Structural beats are load-bearing walls** - can repaint (change words) but cannot remove or move them
3. **"More natural" does NOT always mean better** - architecture intentionally uses patterns that create anticipation

---

## Three Core Mechanics

Every script decision must serve one of these three mechanics.

### Mechanic 1: Orientation (Where Am I?)

| Where | What to do |
|-------|-----------|
| Hook | Set destination + roadmap (2-3 categories, NOT a list of blocks) |
| Block openers | Remind viewer where they are |
| Structural reminders | Every 2-3 minutes, reconnect to main promise |

### Mechanic 2: Anticipation (What's Coming?)

| Element | Cadence |
|---------|---------|
| Forward Pulls | Every 45-60 seconds, tell what's next |
| WHY moments | Create curiosity about implications |

**Pattern:** Tell what's coming -> Make them want it -> Deliver -> Immediately tell what's next

### Mechanic 3: Withholding (Delayed Gratification)

| Phase | % of block | Rule |
|-------|-----------|------|
| SETUP | 30-40% | Present problem, create questions. DON'T hint at solution. |
| TENSION | 40-50% | Show journey to answer. DON'T give action steps yet. |
| PAYOFF | 10-20% | Deliver answer. DON'T over-explain once they get it. |

**DEADLY MISTAKE:** Giving payoff in first 30 seconds of a block.

---

## Forward Pull System

### Hierarchy (Weakest to Strongest)

| Level | Name | Example | Verdict |
|-------|------|---------|---------|
| 1 | Generic Preview | "Next, I'll show you the system" | NEVER USE |
| 2 | Topic Preview | "Next, how to write titles" | NEVER USE |
| 3 | Specific Preview | "why Connor's title is killing his reach" | MINIMUM |
| 4 | Curiosity Gap | "...and it's not what you think" | GOOD |
| 5 | Implied Stakes | "you need to see why they're failing, because if you don't understand this, the tactics won't work" | BEST |

**Minimum standard: Level 3 or higher. Never Level 1-2.**

### Three Types

| Type | Where | Examples |
|------|-------|---------|
| Block Openers | Start of blocks | "So here's the deal..." / "Let's talk about [X]" / Just start with premise |
| Mid-Block | Every 45-60s within blocks | "I'm going to show you..." / "Before I show you how to fix this..." |
| Structural Reminders | Every 2-3 minutes | "That's step one. Step two is where this gets interesting..." |

### BANNED Forward Pull Patterns

- "So that's X. Now let's move to Y..." (signals ending)
- "Now that we've covered X, let's discuss Y..." (too formal)

## WHY Moments

### Two Types

| Type | Pattern | Example |
|------|---------|---------|
| Implication WHY | [Fact]. That means [implication for viewer] | "Only 3% click past page one. That means if you're not there, you don't exist." |
| Causation WHY | [Effect]. This happens because [cause] | "Your reach drops after 48 hours. That's because the algorithm re-scores every video at the 48-hour mark." |

### BANNED WHY Setup Phrases (NEVER use these)

- "And this is why..."
- "And this is critical because..."
- "Here's why this matters..."
- "This is important because..."
- "Here's the thing -"
- "The thing is..."
- "And here's what's happening..."
- "Here's the key insight:"
- "Something magical happens -"

### GOOD WHY Patterns

- "That means [implication]"
- "That's because [reason]"
- "So [consequence]"
- "When you [X], [Y happens]"
- "Which is why [consequence]"
- Just state it directly

### Re-Tension Rule (CRITICAL)

Every WHY moment MUST be followed by new tension within 1-2 sentences.

**Test:** Ask "What's the next question this creates?" Add that before moving on.

## Retention Enhancement Techniques

### Pattern 1: Tension Loop (Every 30-60 seconds)

```
Problem [raise]
-> Solution [tease]
-> But here's why harder [deepen]
-> When you do X, Y happens [release]
-> But avoid this next thing [raise again]
```

### Pattern 2: Visual Storytelling Over Abstraction

"This is like [metaphor]. Imagine [visual scene]. When [action happens], [visual result]. That's exactly what happens when [connection to concept]."

### Pattern 3: Strategic Overexplaining

Before every major Payoff, ask: "Do they know WHY this matters?" If not, add ONE sentence.

### Pattern 4: Brain Funnel (Strip Non-Essential)

**"LAND AND MOVE" test:** Find the sentence where the point lands. Everything after it is probably a cut.

### Pattern 5: Credibility Through Vulnerability

```
Struggle (with specifics) -> Turning point -> Discovery -> Authority
```

### Pattern 6: Specificity Over Generality

Never generalize what architecture made specific. Use exact numbers.

### Pattern 7: Expectation Management

"[What we just covered] + [Where we are] + [What's coming and why you should care]"

## Setup-Tension-Payoff Execution

### Preferred Teaching Pattern: Problem -> Story -> Action

| Phase | Role | Rule |
|-------|------|------|
| Setup (Problem) | Introduce problem without solving it | No hints at the answer |
| Tension (Story) | Journey, failures, breakthrough | No action steps yet |
| Payoff (Action) | What they should do now | Don't over-explain once it lands |

### Payoff Protection

Before writing a block, identify the answer. If it appears in Setup or Tension, RESTRUCTURE.

### Lists Within Blocks

Still use Setup-Tension-Payoff for EACH item in a list.

**BAD:**
> "First - stop comparing yourself. Here's why..."

**GOOD:**
> "First, there's something most creators do that guarantees they'll quit. [Setup] When you're stuck at 50 views... [Tension] That's why you need to stop comparing. [Payoff]"

## Language Rules

### Style

- 5th grade reading level
- Contractions everywhere
- Short punchy sentences (see thresholds below)
- Write TO them, not AT them

### Sentence Length Thresholds

| Words | Action |
|-------|--------|
| Over 25 | MUST split |
| Over 20 | SHOULD split |
| 3+ consecutive over 15 | Rhythm is flat - vary sentence length |

Every block needs at least one **staccato sequence**: 3+ sentences under 8 words each.

**Good:** "Month one, 10. Month three, 30. Month six, 50. Five thousand a month."
**Bad:** A paragraph where every sentence is 12-18 words with no variation.

### Question Fragments - ABSOLUTELY BANNED

The `[Subject]? [Answer]` pattern is banned in all forms.

| BANNED | REPLACEMENT |
|--------|-------------|
| "That video? It blew up." | "That video blew up." |
| "The result?" | Just state the result. |
| "[Thing]? [Statement about thing]." | Just make it a statement. |

### Natural Phrases to Use

- "Here's what..."
- "Here's the deal..."
- "So let me show you..."
- "Check this out..."
- "Look at this..."

### Phrases to AVOID

- "Here's the thing -"
- "At the end of the day"
- "The bottom line is"
- "Game-changer"
- "So those are the [X things]..."
- "That's the game"
- "Something magical happens -"

## CTA Integration Rules

### Placement

| CTA Type | Placement | Count |
|----------|-----------|-------|
| Lead magnet | Early Block 1 + later block before final CTA | 2 mentions required |
| Sponsor (if applicable) | END of Block 1, own STP mini-block | 1 |
| Like request | After first major value delivery | 1 |
| Final CTA | Last block | 1 |

**Minimum 140 words between any two CTAs.**

### Final CTA Structure

| Step | What it does | Example pattern |
|------|-------------|----------------|
| Insight | Direct statement about what they've gained | "You now know [X]." |
| Gap | What they still need + why it matters | "But [what they still need]. [Why it matters]." |
| Bridge | Where to go + what they'll get + action | "[Where to go]. [What they'll get]. [Action]." |

### NEVER Signal Ending

- NEVER say "So those are the [X things]..."
- NEVER say "That's the game"
- NEVER use any phrasing that tells the viewer the video is wrapping up

## Production Blueprint Format

Every script must include these production elements.

### Dialogue Format

Use `| Visual | What You Say |` tables for all spoken content.

### Production Markers

| Marker | Purpose |
|--------|---------|
| `LAYOUT:` | Speaker layout (speaker_full, split_right, split_5050, etc.) |
| `SLIDE XX:` | Slide reference for visual |
| `SCREEN XX:` | Screen recording reference |
| `BROLL:` | B-roll description or Pexels search query |
| `GIF:` | GIF description or Giphy search query |
| `SFX:` | Sound effect trigger |
| `TEXT:` | Text overlay content |

### Required Asset Lists

- Screens list (if any screen recordings)
- Slides list with descriptions
- Pexels B-roll queries
- GIF queries
- SFX map (which sounds, where)
- Text overlay list
- Layout flow section for timeline builder

### Word Count

Calculate target duration at 217 WPM baseline.

**CRITICAL: Word counts refer to SPOKEN DIALOGUE ONLY.** Count only the text inside quotes in the "What You Say" column. Do NOT count layout markers, slide descriptions, SFX cues, text overlay labels, asset lists, layout flow sections, table headers, section headings, or any other non-spoken text. Production markers are NOT spoken words.

**Self-check:** After writing the script, extract ONLY the quoted dialogue text and count those words. That number must hit the target below. If it doesn't, the script is short and needs more spoken content.

| Target Duration | Spoken Word Count |
|----------------|-------------------|
| 5 minutes | ~1,085 words |
| 8 minutes | ~1,736 words |
| 10 minutes | ~2,170 words |
| 12 minutes | ~2,604 words |
| 15 minutes | ~3,255 words |

## Editorial Tightening Rules

Every sentence must earn its place. When reviewing or editing any script, apply these.

### Cut These Patterns

**Resume recitation.** "I've been building since I was a teenager, co-founded an agency, grew to six figures." Credibility comes from the story you're telling, not a list of achievements. Show what you built and what happened. One sentence of background is enough.

**Vague bridges.** "That should tell you something about what customers actually value." / "And knowing X is one thing..." / "And that's where it gets interesting." These sound meaningful but say nothing. Either state the specific thing or cut the sentence.

**Redundant elaboration.** If the point already landed, don't restate it in different words. "SaaS compounds" followed by "SaaS revenue is recurring" is the same insight twice. Apply the LAND AND MOVE test: find the sentence where the point lands. Everything after it that says the same thing is a cut.

**Wrong-product pitches.** In a VSL for a community, don't spend 47 words describing your SaaS product's features. One sentence establishes proof. Anything more is selling the wrong thing.

**"Let me show you" openers.** Don't ask permission to teach. "Let me show you some math" is soft. "Every month you don't launch costs way more than you think" is a claim with stakes. State the claim directly.

### Claim Integrity

Bold claims must be paid off. If you say "designed to keep them stuck" (implies someone intentionally designed it that way), you must explain WHO and WHY later in the script. If you can't pay it off, soften: "misses the one thing that actually matters." Conspiracy-adjacent claims without payoff erode trust.

### Tense Shifts Need Signaling

When shifting from past tense to counterfactual ("I built DM Champ while running an agency" then "A community of builders around me. Coaching calls every week"), the listener loses track of what's real vs hypothetical. Signal the shift: "If I'd had this back then?" Without the signal, it sounds like you're describing something that happened.

### Dangling Referents

"And it always gets hard" - what's "it"? Make the referent explicit: "And building always gets hard."
"That's the gap" - what gap? A half-listening viewer missed the referent. Say "That's what's missing" instead.

Every pronoun and "it/this/that" should have an obvious referent within the same sentence or the one immediately before it. If you have to think about what "it" refers to, rewrite.

### Narrative Consistency

If the script's story is "I built this alone and wish I'd had help," every line must hold that up. Watch for lines that accidentally imply the founder HAD the help they're selling. "One conversation about pricing was worth more than every video" implies someone coached you. But if your whole pitch is "that coach didn't exist, so I built it," you just contradicted yourself. Flip it: "I watched every pricing video. None could tell me what to charge." Same emotional beat, no contradiction.

## Chapter Re-Hook Rules

Every block opener must work as a standalone entry point. Viewers scrub to chapters and make a stay-or-leave decision in 2-3 seconds.

### The Cold Landing Test

Read the first 2-3 sentences of each block. Pretend you skipped the entire video and landed HERE. Would you stay?

**PASS examples:**
- "Most SaaS communities sell you a course and disappear." (bold claim, immediate differentiation)
- "'I don't even have an idea yet.' That's the number one thing people tell me." (viewer's own objection)
- "Every month you don't launch costs way more than you think." (specific claim with stakes)

**FAIL examples:**
- "Let me show you some math that most people never think about." (soft, asks permission)
- "So now let's talk about the community." (continuation, not standalone)
- "Moving on to pricing." (label, not a hook)

### Re-Hook Requirements

- Opens with: bold claim, surprising stat, viewer's own objection, or pattern interrupt
- NEVER opens with: "So now let's...", "Moving on to...", "Let me show you...", "Alright so...", "Next up..."
- Must include visual reset in production markers: SFX whoosh + layout change or new slide within first 3 seconds
- Chapter title in the CHAPTER MAP must also be a curiosity gap, not a label ("Every Month You Don't Launch" not "The Math")

## Story Overlap Prevention

Every script must use unique stories. Viewers watch multiple videos and notice when the same anecdote appears twice.

### Before Writing

1. Read all scripts from `youtube/weekly-production/*/script.md` (most recent 6-8 videos)
2. Extract every personal story, example with specific numbers, and named project reference
3. Cross-reference against the current architecture's story beats

### What Counts as Overlap

- Same specific anecdote told in two videos (even if framed differently)
- Same example with the same numbers (e.g., "$100/day API costs" in two videos)
- Same named project as proof for the same point
- Same metaphor or analogy

### What Does NOT Count

- Mentioning DM Champ as revenue proof in one video and as a technical example in another
- Two videos both discussing "costs" but with completely different stories
- Brief references to the same project for context (not as a featured story)

### If Overlap Is Found

STOP. Flag the overlap to the user/team lead with: "This story was used in [video]. Need a replacement story that makes the same point." Do NOT rewrite around it yourself - the user provides real replacement stories.

## Story Authenticity and Fact-Checking

### Never Fabricate

Every story, stat, and specific claim must come from one of:
1. The user (via architecture's verified brain dump)
2. Web research (with source)
3. The story inventory (`youtube/stories/inventory/`)

If a beat says "tell the debugging story" but the architecture has no specifics, write `[NEEDS REAL DETAILS FROM USER]` inline. Do not invent dollar amounts, timeframes, character names, or outcomes.

### Fact-Check Claims

Before writing a stat or claim into dialogue:
1. Check if the architecture lists a source
2. If no source, use WebSearch to verify
3. If unverifiable, mark as `[UNVERIFIED: source needed]`

Real stories have jagged, specific details. Round numbers ("about a thousand"), vague timeframes ("a few months"), and generic descriptions ("some developer") are red flags for fabrication.

### Web Research During Writing

The script-writer has access to WebSearch and WebFetch. Use them to:
- Verify stats and claims before writing them into dialogue
- Find current data to make the script feel timely
- Check if a referenced tool, framework, or event is still current

Do NOT use web research to add content the architecture didn't plan for. The architecture is locked. Web research is for verification and enrichment, not scope expansion.

## Common "Improvements" That Break Things

| Trap | What sounds right | What it actually does |
|------|-------------------|----------------------|
| "Too wordy, simplify" | Tighten prose | Separated preview from stakes |
| "Repetitive, cut it" | Remove repeated ideas | Removed WHY moment AND re-tension |
| "Too salesy, soften" | Tone down language | Removed stakes ("might be affecting" vs "costing 80%") |
| "Better flow" | Reorganize for readability | Revealed answer too early |

### When You CAN Improve

- Remove genuine redundancy (saying same thing twice with no structural purpose)
- Fix awkward phrasing while keeping function
- Add natural connectors between beats
- Tighten after point lands (cut belaboring)

## Validation Checklist (MANDATORY - Gate Before Output)

**This validation is NOT optional. The script writer MUST run this before outputting the final script. If any check fails, fix the script first.**

### Step 1: WHY Moment + Re-tension Audit (MUST SHOW WORK)

For every WHY moment in the script, list it in a table:

| WHY Moment (quote) | Block | What follows within 1-2 sentences? | Re-tension present? |
|---------------------|-------|-------------------------------------|---------------------|

If ANY row shows "No" for re-tension, add a new question/curiosity gap within 1-2 sentences BEFORE moving on. Test: "What's the next question this creates?" If nothing, re-tension is missing.

**Do NOT output the script until every row shows re-tension present.**

### Step 2: Per Block

- [ ] All beats from architecture used in order
- [ ] Payoff withheld until end of block
- [ ] Problem -> Story -> Action pattern followed
- [ ] Forward Pulls every 45-60s at Level 3+
- [ ] WHY moments every 30-60s with natural phrasing
- [ ] Re-tension after every WHY moment (verified in Step 1)
- [ ] No banned phrases or question fragments
- [ ] Contractions throughout
- [ ] Points land and move on
- [ ] Tension loops complete (Raise -> Tease -> Deepen -> Release -> Re-tension)
- [ ] Stories follow vulnerability-first pattern
- [ ] All specific numbers present

### Step 3: Full Draft

- [ ] Delivers on title/thumbnail promise
- [ ] Could record with minimal rewriting
- [ ] Forward Pull every 45-60s throughout ENTIRE video
- [ ] No AI phrases from avoid list
- [ ] CTA placements natural with 140+ word spacing
- [ ] Blocks flow naturally
- [ ] No ending signals before final CTA

## Success vs Failure

| | Success | Failure |
|--|---------|---------|
| Voice | Creator reads it and says "this sounds like me" | Creator has to rewrite paragraphs |
| Tone | Conversational, spoken feel | Sounds like a lecture or blog post |
| CTAs | Feel natural, woven into content | Feel jarring, break flow |
| Anticipation | Each block creates anticipation for next | Payoffs given too early |
| Pacing | Forward Pulls every 45-60s throughout | 90+ seconds without a Forward Pull |
| Rewrite needed | Minor tweaks only | Structural rewrite required |
